### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| **扫描** | 每日收盘后全市场扫描，识别逆流信号股票 |
| **入池** | 将符合条件的股票加入观察池 |
| **出池** | 将不再符合的股票移出池子 |
| **分析** | AI深度分析候选股票，确认中期行情逻辑 |
| **记录** | 维护股票池历史，记录入池/出池原因 |

### 1.3 与现有模块的关系

| 模块 | 关系 | 说明 |
|------|------|------|
| 信号模块 | 下游 | 选股模块负责填充股票池，信号模块负责判断操作 |
| 自进化模块 | 支撑 | 选股效果纳入自进化统计，反馈优化参数 |
| 飞书推送 | 触发方 | 选股结果通过飞书推送 |

### 1.4 边界说明

- ✅ **做**：维护股票池，为信号模块提供候选标的
- ❌ **不做**：交易决策（信号模块的职责）
- ❌ **暂不做**：信号模块的交易逻辑调整（待选股模块稳定后再说）

## 二、运行原理

### 2.1 核心理念

**逆流选股法的本质：发现主力悄悄吸筹的痕迹**

> 市场看到的是资金在流出、价格在震荡
> 看透本质是：主力在拉升时出货、砸盘时吸筹
> 两边都让对手盘亏钱，自己慢慢收集筹码

### 2.2 逆流信号识别

#### 阶段一：量化扫描（全市场）

**核心逻辑：**
```
近10日累计大单净流出金额较大
    +
价格区间没有明显下跌（坚挺）
    +
底部逐步抬高（支撑有效）
    =
疑似主力在"拉高卖、砸下去接"的循环吸筹
```

**具体量化条件：**

| 条件 | 阈值 | 说明 |
|------|------|------|
| 近10日累计大单净流出 | > X万 | 待回测确定（候选：5000万/8000万/1亿）|
| 10日价格区间涨跌幅 | > -5% | 价格坚挺，不随流出而跌 |
| 底部抬高 | 近10日最低 > 30日前最低 | 支撑位逐步上移 |
| 近10日振幅 | < 15% | 控盘稳定，非无序波动 |
| 均线趋势 | MA20之上 | 整体趋势未破坏 |
| 日均成交额 | > 5000万 | 流动性要求 |

#### 阶段二：AI公告分析（候选池）

**目的：** 找到"为什么会涨"的逻辑支撑，过滤假信号

**分析维度：**

| 维度 | 关注点 | 风险标记 |
|------|--------|----------|
| 减持进度 | 是否接近尾声 | 利好/利空 |
| 业绩预告 | 是否超预期 | 利好/利空 |
| 行业政策 | 是否受益 | 利好 |
| 资金用途 | 募资投向热门方向 | 利好 |
| 高管动态 | 是否在增持 | 利好 |
| 限售解禁 | 解禁时间/规模 | 利空 |

**AI分析输出：**
```
股票代码: XXXXXX
AI分析结论: [现象本质解读]
中期行情概率: 高/中/低
入池建议: 建议入池/不建议入池
入池理由: [具体分析]
```

#### 阶段三：入池决策

```
量化扫描通过
    +
AI分析确认（概率高/确定性高）
    ↓
进入观察池（≤10只）
```

### 2.4 出池标准

| 出池条件 | 说明 | 优先级 |
|----------|------|--------|
| 超时无信号 | 入池 > 60日无有效信号 | P1 |
| 触发止损 | 价格跌破关键支撑 | P0 |
| 信号消失 | 连续20日无逆流信号 | P1 |
| 基本面恶化 | 业绩预亏/重大利空公告 | P0 |
| 趋势破坏 | 有效跌破MA20且3日内未收复 | P1 |
| AI提示风险 | AI分析发现重大风险 | P0 |

### 4.2 核心类设计

#### 逆流Selector

```python
class 逆流Selector:
    """逆流选股法核心类"""

    def __init__(self, data_engine):
        self.engine = data_engine

    def scan(self, days=10) -> list[str]:
        """
        全市场扫描，返回逆流信号候选股票
        返回: list[stock_code]
        """

    def calculate_flow_metric(self, code, days=10) -> dict:
        """
        计算个股资金流指标
        返回: {
            '累计大单净流出': float,
            '区间涨跌幅': float,
            '底部抬高幅度': float,
            '振幅': float,
            'MA20趋势': str
        }
        """

    def is_niux_flow(self, metrics) -> bool:
        """
        判断是否符合逆流条件
        返回: True/False
        """
```

#### 池管理器

class PoolManager:
    """股票池管理器"""

### 4.3 运行流程

```python
# main.py 日模式新增部分
def daily_mode():
    # 1. 选股模块
    selector = 逆流Selector(data_engine)
    pool_manager = PoolManager(pool_dir)

    # 扫描候选
    candidates = selector.scan(days=10)
    print(f"扫描到 {len(candidates)} 只候选股票")

    # AI分析
    ai_analyzer = AIAnalyzer(llm_client)
    confirmed = []
    for code in candidates:
        analysis = ai_analyzer.analyze(code)
        if analysis['入池建议'] == '建议':
            confirmed.append((code, analysis))
        if len(confirmed) >= 10:  # 最多10只
            break

    # 入池
    for code, analysis in confirmed:
        pool_manager.add_to_observation(code, "逆流信号", analysis)

    # 出池检查
    pool_manager.check_exit()

    # 2. 信号模块（现有）
    signals = run_signal_module(pool_manager)

    # 3. 推送
    send_notification(pool_manager, signals)
```
