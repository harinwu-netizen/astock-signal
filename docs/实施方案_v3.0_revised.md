# 信号灯 v3.0 实施方案（修订版）

> **版本**：v3.0-revised  
> **基于**：原始方案 + 评估反馈  
> **原则**：砍掉高风险模块，聚焦核心价值，快速落地  
> **发布日期**：2026-03-30

---

## 一、修订说明

### 砍掉的内容
| 模块 | 原因 |
|------|------|
| AI 参数调优 | 过拟合风险大，循环依赖问题 |
| 移动端适配 | Web 做好足够，移动端投入产出比低 |
| Redis 缓存 | 文件缓存短期足够，无需引入运维复杂度 |
| 市场风格自适应 | 信号系统尚未稳定，先不做动态权重 |

### 调整的内容
| 模块 | 调整 |
|------|------|
| 回测模块 | 先做**单股60日**，不做全量股票池回测 |
| 实盘对接 | **默认人工确认**，不做全自动模式 |
| 工作量预估 | 基础版 **3-4周**（原1-2周偏乐观） |

#### 任务 1.4：尾盘开仓风险强化过滤
**文件**：`trading/pre_check.py`（修改）  
**依赖**：任务1.3  
**耗时**：1天

**新增3层过滤**：

**4.1 量能承接检查**
```python
# 量比 0.8-1.5，尾盘5分钟量逐步放大
def check_volume承接(code) -> bool:
    volume_ratio = 当日成交量 / 日均成交量
    if not (0.8 <= volume_ratio <= 1.5):
        return False
    # 尾盘5分钟量 > 开盘前5分钟量
    return recent_5min_vol > early_5min_vol
```

**4.2 板块共振检查**
```python
# 板块MA5多头 + 板块涨幅≥0.5%
def check板块共振(code) -> bool:
    block_ma5_ok = 板块MA5多头排列
    block_change >= 0.5
    return block_ma5_ok and block_change_ok
```

**4.3 个股位置检查**
```python
# 站在MA60上方，距阶段高点回撤≤10%
def check个股位置(code, current_price) -> bool:
    ma60_ok = current_price > MA60
    drawdown = (phase_high - current_price) / phase_high
    return ma60_ok and drawdown <= 0.10
```

**大盘判断优化**：
- 新增创业板/科创板联合判断
- 弱势市场定义更精细（创业板+科创板双跌才算弱势）

**验收标准**：
- 三层过滤全部通过才允许尾盘开仓
- 任一层失败记录具体原因

class BacktestEngine:
    def run(self, code: str, days: int = 60,

**验收标准**：

## 八、风险与对策

| 风险 | 等级 | 对策 |
|------|------|------|
| 回测结果与实盘差异大 | 高 | 先模拟交易3个月，不急着上实盘 |
| 数据源切换漏消息 | 中 | 切换后补全数据，禁止切换过程中交易 |
| AI预警疲劳 | 低 | 分级推送（只推重要预警） |
| 成本计算遗漏 | 中 | 用真实交割单核对 |
