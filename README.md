## 1. 设计思路

### 1.1 核心理念

信号灯是一套**轻量级A股量化交易系统**，不追求高频和复杂策略，而是基于技术指标信号驱动，在**弱势/强势/震荡**三种市场环境下自适应切换交易逻辑，追求**稳定胜率 + 严格风控**。

### 1.2 核心设计原则

- **有信号才交易**：没有买卖信号时保持空仓，不主观预测方向
- **市场自适应**：根据大盘状态自动切换专属信号系统
- **三层架构**：弱市反弹、强势趋势、震荡波段三个完全独立的信号子系统
- **信号驱动退出**：卖出决策由信号系统主导，安全网兜底
- **AI增强**：每次成交后调用大模型分析，提供独立客观的决策参考

### 1.3 市场状态定义

| 状态 | 中文 | 判断依据 | 核心策略 |
|------|------|---------|---------|
| `WEAK` | 弱势 | 任意指数近5日跌幅超1.5% | 严格风控，快进快出，RSI<35超跌才买，让利润跑 |
| `STRONG` | 强势 | 上证/创业板/科创50近5日加权涨幅 > 0，大盘趋势向上 | 趋势跟踪，MA20跟踪止损，让利润奔跑 |
| `CONSOLIDATE` | 震荡 | 不满足上述条件，方向不明 | 波段操作，RSI低位拐头买，RSI/布林上轨止盈 |

## 3. 三层信号系统

### 3.1 弱市反弹系统（signal_weak.py）

适用于大盘下跌趋势中的超跌反弹机会。

**买入信号（满足2/3个）：**
| 指标 | 条件 | 含义 |
|------|------|------|
| RSI超跌 | RSI(6) < 35 + 拐头向上 | 真正超跌，不是半山腰 |
| 触及布林下轨 | 收盘价 ≤ 布林下轨 × 1.01 | 获支撑 |
| 缩量整固 | 量比 < 0.6 | 抛压轻 |

**卖出信号：**
| 条件 | 含义 |
|------|------|
| RSI > 65 | 反弹到位，强制止盈 |
| 触及布林上轨 | 触及区间上沿 |
| ATR追踪止损 | 移动止损，锁定利润 |

**止损策略：** 硬止损 -2%，弱势不抗单。

---

### 3.2 强市趋势系统（signal_strong.py）

适用于大盘上涨趋势中的趋势跟踪机会。

**买入信号（满足2/3个）：**
| 指标 | 条件 | 含义 |
|------|------|------|
| MA多头排列 | MA5 > MA10 > MA20 | 上升趋势 |
| MACD扩散 | DIF向上且柱状体放大 | 动能增强 |
| 量价齐升 | 成交量放大 + 股价创新高 | 资金推动 |

**卖出信号：**
| 条件 | 含义 |
|------|------|
| MA5下穿MA10 | 短期趋势反转 |
| 收盘跌破MA20 | 中期趋势破坏 |
| RSI > 75 | 超买预警 |

**否决过滤器：** 全局RSI(14) > 60时不追强势突破（熊市不追强，避免被套）。

**止损策略：** 跌破MA20 或 亏损>8%。

---

### 3.3 震荡市波段系统（signal_consolidate.py）

适用于大盘方向不明时的区间震荡行情。

**买入信号（满足2/3个）：**
| 指标 | 条件 | 含义 |
|------|------|------|
| RSI低位拐头 | RSI(6) 在35-50区间且拐头向上 | 底部整固完成 |
| 回踩均线 | 收盘价 ≤ MA10 × 1.01 | 获均线支撑 |
| 缩量整固 | 量比 < 0.7 | 抛压轻 |

**卖出信号（满足2/3个）：**
| 指标 | 条件 | 含义 |
|------|------|------|
| RSI回落 | 买入后RSI从反弹高位再次回落 | 反弹失败 |
| 触及布林上轨 | 收盘价 ≥ 布林上轨 × 0.98 | 触及区间上沿 |
| RSI>55 | RSI进入偏高区域 | 反弹到位 |

**止损策略：** 布林下轨 - 1.5×ATR，或 亏损>10%。

---

### 3.4 统一路由层（signal_unified.py）

三层信号的智能调度器：

```
市场状态判断
    │
    ├── WEAK        → 调用 signal_weak（弱市反弹）
    ├── STRONG      → 调用 signal_strong（强市趋势）
    └── CONSOLIDATE → 调用 signal_consolidate（震荡波段）
    │
    └── 持仓期间：sell_count > 0 → 直接触发 SELL（跨系统否决）
```

**跨系统否决逻辑：**
- 震荡市：不追强市趋势破坏的票（避免逆势）
- 弱势/强市：主要看自己的系统，否决逻辑更宽松

## 4. 工作流程

### 4.1 每日监控流程

```
定时触发（每10分钟）
    │
    ▼
┌──────────────────────────────────┐
│ 1. 数据获取                       │
│    获取股票池实时行情 + 30日K线   │
└──────────────┬───────────────────┘
               ▼
┌──────────────────────────────────┐
│ 2. 大盘状态判断                   │
│    检测上证/创业板/科创50近5日涨跌 │
│    判断：强势 / 弱势 / 震荡        │
└──────────────┬───────────────────┘
               ▼
┌──────────────────────────────────┐
│ 3. 三层信号扫描                   │
│    - 持仓股：传入真实买入价       │
│    - 候选股：正常扫描             │
│    输出：decision + 信号计数      │
└──────────────┬───────────────────┘
               ▼
┌──────────────────────────────────┐
│ 4. 统一路由决策                   │
│    - 优先使用当前市场子系统       │
│    - 跨系统否决过滤               │
│    - 决策：BUY / WATCH / SELL    │
└──────────────┬───────────────────┘
               ▼
┌──────────────────────────────────┐
│ 5. 风控检查                       │
│    - PreCheck: 涨跌停/停牌/流动性 │
│    - RiskControl: 大盘过滤/黑天鹅 │
│    - 锁仓期检查（连续亏损股票）   │
└──────────────┬───────────────────┘
               ▼
┌──────────────────────────────────┐
│ 6. 交易执行（notify_only模式）    │
│    推送信号到飞书，等待用户确认   │
└──────────────┬───────────────────┘
               ▼
┌──────────────────────────────────┐
│ 7. AI增强分析（异步）             │
│    MiniMax大模型分析交易逻辑      │
│    完成后再次飞书推送             │
└──────────────────────────────────┘
```

### 4.2 持仓管理流程

```
持仓中（每日更新）
    │
    ├── 触发 ATR 追踪止损 ────────────→ 止损卖出（信号驱动）
    ├── 触发 固定止损 ─────────────────→ 止损卖出
    ├── 弱势市场：RSI > 65 ───────────→ 强制止盈
    ├── 弱势市场：触及布林上轨 ────────→ 波段止盈
    ├── 震荡市：RSI > 55 或触及布林上轨 → 波段止盈
    ├── 强势市场：MA5下穿MA10 ─────────→ 趋势跟踪卖出
    ├── 持仓超期（震荡7天/强市30天）──→ 超期平仓
    └── 每日：更新持仓浮动盈亏 ────────→ 记录
```

## 5. 功能说明

### 5.1 信号计算（SignalCounter / signal_unified）

**`count_signals(klines, realtime, market_regime, buy_price=None)`**

对单只股票计算完整信号体系：

| 分类 | 字段 | 说明 |
|------|------|------|
| 经典信号 | `buy_count` | 买点信号触发数量（10个指标） |
| 经典信号 | `sell_count` | 卖点信号触发数量（6个指标） |
| 震荡信号 | `consolidate_buy_count` | 震荡市波段买点触发数量（3选2） |
| 震荡信号 | `consolidate_sell_count` | 震荡市波段卖点触发数量（3选2） |
| 决策 | `decision` | BUY / WATCH / SELL / STOP_LOSS / TAKE_PROFIT |
| 仓位 | `position_ratio` | 建议仓位 0%~100% |
| 指标 | `rsi_6`, `atr`, `ma5/10/20`, `bb_upper`, `macd_dif` | 关键技术指标值 |

**关键：持仓股必须传入真实 `buy_price`，否则信号系统无法区分持仓/观望状态。**

### 5.2 市场状态判断（MarketFilter）

基于上证指数、创业板指、科创50三个指数的5日均线综合判断：

```
强势：3个指数近5日加权涨幅 > 0
弱势：任意1个指数近5日跌幅 > 1.5%
震荡：其他所有情况
```

### 5.3 动态止损（ATR）

ATR（Average True Range）反映近期股价波动幅度，用于计算动态止损位：

```
止损位 = max(买入价 - 2×ATR, 买入价 × 0.95)
移动止损：持仓期间持续上浮，永不下降
```

### 5.4 仓位管理

| 市场 | 条件 | 仓位 |
|------|------|------|
| 弱势 | RSI<35 + 满足2/3信号 | 100% |
| 震荡 | consolidate_buy_count = 3 | 100% |
| 震荡 | consolidate_buy_count = 2 | 50% |
| 震荡 | 其他 | 0% |
| 强势 | 满足2/3信号 + RSI<60 | 100% |
| 强势 | 满足2/3信号 | 50% |

最大同时持仓：**3只股票**

### 5.5 交易执行（Executor）

支持两种模式：

| 模式 | notify_only | 说明 |
|------|-------------|------|
| 模拟交易 | `true` | 推送信号到飞书，人工确认后执行 |
| 实盘交易 | `false` | 自动执行（需配合券商接口） |

### 5.6 AI增强分析（LLMAnalyzer）

每次成交后自动触发，调用 MiniMax 大模型分析：

**买入分析：** 交易逻辑是否合理、风险点、后市操作建议
**卖出分析：** 卖出理由是否正确、是否会卖飞、是否需要回补

### 5.7 飞书通知（FeishuNotifier）

| 类型 | 触发条件 | 内容 |
|------|---------|------|
| 信号报告 | 每10分钟扫描 | 股票池决策、信号数、价格 |
| 交易通知 | 每次成交 | 股票代码、价格、数量、金额、原因 |
| AI分析 | 成交后异步 | 大模型输出的交易分析意见 |
| 持仓日报 | 每日收盘 | 当日持仓、浮动盈亏、信号变化 |
| 紧急预警 | 亏损>8% | 亏损超限警告 + AI分析 |
| 大盘异动 | 跌幅>2% | 多指数监控 + 减仓建议 |

## 10. 故障排除

### 10.1 常见问题

**Q: 启动报错 `ModuleNotFoundError: No module named 'txstock'`**
```
解决：pip install txstock 或确认 skills/txstock/scripts/ 目录存在
```

**Q: 飞书通知发不出去**
```
检查1：OpenClaw Gateway 是否运行
检查2：pending_messages.json 是否有积压消息
检查3：scripts/send_pending.py 服务是否启动
解决：systemctl restart astock-signal-send
```

**Q: 买入失败 `资金不足`**
```
原因：.env 中 TOTAL_CAPITAL 配置过小
解决：编辑 .env，设置 TOTAL_CAPITAL=1000000
```

**Q: AI分析不返回结果**
```
原因：LLM_API_KEY 无效或网络问题
日志：查看 logs/ 目录下的错误信息
```

**Q: 数据获取失败**
```
原因：腾讯财经接口不可用
解决：等待网络恢复，系统会自动重试
```
